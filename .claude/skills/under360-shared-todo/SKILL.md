# Under360 — Shared TODO protocol

**ปัญหาที่แก้:** โปรเจคนี้รันหลาย session ขนานกัน (m-track, u-track, a-track, เอิธ) ทุก session ชอบ ping/ส่งบรีฟให้นัทอ่าน → **นัทกลายเป็นคอขวด ต้องอ่านทุกอย่าง** · แก้ด้วย **TODO.md กลางไฟล์เดียว** ที่ repo root

## เมื่อไหร่ใช้
- **ตอนเริ่ม session** → อ่าน `TODO.md` ก่อน เพื่อรู้ว่ามีอะไรค้าง/ใครขออะไร
- **ทุกครั้งที่จะ:** จดงานค้างของตัวเอง · ส่งต่องานให้ track อื่น · ขอให้นัทไปทำอะไร (สร้าง project, รัน SQL, กดปุ่มในคอนโซล, เทส ฯลฯ)
- **แทนการ ping/send_message/บรีฟ** — ยกเว้นเรื่อง **ด่วนจริง** (บล็อกงาน, ต้องตอบเดี๋ยวนั้น) ค่อย ping

## วิธีเขียน
1. **pull ก่อนแก้เสมอ** (กันชนกับ session อื่น)
2. **แก้เฉพาะ section ของ track ตัวเอง** (🟢 M / 🔵 U / 🟣 A-เอิธ) — ห้ามแก้ section คนอื่น
3. เพิ่มงานที่ **นัทต้องทำ** ลงหัวข้อ **"🔴 นัทต้องทำ"** ได้ทุก track (ต่อท้ายด้วย `[track]` ว่าใครขอ)
4. `- [ ]` ค้าง · `- [x]` เสร็จ · เสร็จแล้วขยับไป "✅ เพิ่งเสร็จ" หรือติ๊ก [x]
5. commit เฉพาะ TODO.md (+ ไฟล์ตัวเอง) · push · **pull ก่อน push**

## หลักการ
- **นัทเปิด TODO.md เมื่อไหร่ก็ได้** ตามจังหวะตัวเอง — ไม่ถูก interrupt ตลอด
- นัทดูแค่ **"🔴 นัทต้องทำ"** ก็เห็นทุกอย่างที่อยู่บนบ่าตัวเอง จากทุก track ที่เดียว
- แต่ละ track ยังมีลิสของตัวเองในไฟล์เดียวกัน (ใครลิสมัน)
- เกี่ยวโยง `MKT_HANDOFF.md` (รายละเอียด m-track), `CLAUDE.md` (u ดูแล master)
