# Under360 — Session Start Protocol

**กฎเหล็ก: ห้ามแก้โค้ด/ตอบเรื่อง state ของระบบ ก่อนอ่าน context ครบ**

## 1. อ่านไฟล์ context ก่อนเสมอ (ตามลำดับ)
1. `CLAUDE.md` (root ของ repo) — single source of truth
2. `UNDER360_MASTERNOTE_v*.md` (ถ้ามีในโฟลเดอร์ / นัทแนบมา) — เอาเวอร์ชันสูงสุด
3. `UNDER360_HISTORY.md` — timeline ความสำเร็จ + เลขเวอร์ชันปัจจุบัน (u/m/a/doc)

ถ้าไม่เจอไฟล์เหล่านี้ → **ถามหา 1 ครั้ง แล้วหยุดรอ** อย่าเริ่มงานโดยเดา

## 2. สรุปให้นัทดู 5 บรรทัด ก่อนเริ่ม
- เวอร์ชันปัจจุบัน: u? / m? / a?
- งานที่ค้างอยู่ 3 อันดับแรก
- SQL ที่ยังไม่ได้รันใน Supabase (ถ้ามี)
- ไฟล์ที่ยังเป็น mock data
- คำถาม: "วันนี้จะทำข้อไหน?"

## 3. บริบทตัวคน (สำคัญกับวิธีตอบ)
- นัท **ไม่ใช่ developer** — อ่านโค้ดได้บ้าง แต่ไม่เขียนเอง
  → อธิบายเป็นภาษาคน ไม่ใช่ jargon · ให้ผลลัพธ์ที่รันได้จริง ไม่ใช่ทฤษฎี
- **ตอบภาษาไทย** เสมอ (technical term เป็นอังกฤษได้)
- **"ปรับได้เลย" = ทำเลย ห้ามถามซ้ำ**
- ถามกลับได้เฉพาะเมื่อเป็น **fork ระดับ architecture** เท่านั้น
- นัทส่งข้อความ multi-part ในทีเดียว → **ตอบให้ครบทุก part** อย่าตกข้อ
- ถ้า ambiguous → เลือกทางที่ง่ายที่สุด ทำไปก่อน แล้วแจ้งทีหลัง

## 4. ปิด session ทุกครั้ง
- สรุป: ทำอะไรไปแล้ว / next step คืออะไร
- เคาะเลขเวอร์ชัน + เรียกใช้ skill `under360-masternote`
